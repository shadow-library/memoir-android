```bash
# Build debug APK
./gradlew assembleDebug

# Build release APK
./gradlew assembleRelease

./gradlew installDebug

./gradlew test

./gradlew clean
```

## Architecture

Clean Architecture with three layers. Dependencies only flow inward: `ui → domain ← data`.

### `data/`
Talks to all external systems: Firestore, Health Connect, Gemini Vision API, Frankfurter FX API, and EncryptedSharedPreferences.
- Repositories expose `suspend fun` or `Flow` — never raw Firestore types to callers
- All Firestore-to-domain mapping happens inside repositories
- `FirebaseManager.kt` handles dynamic Firebase initialisation (BYOF model — user supplies their own Firebase project credentials)
- `EncryptedConfigStore.kt` is the sole reader/writer of `EncryptedSharedPreferences`
- FX rates are cached in-memory inside `FxRateService` — never written to disk

### `domain/`
Pure Kotlin — zero Android or Firebase imports.
- `model/` — plain data classes, no logic, no annotations
- `usecase/` — one class per use case, one public `suspend operator fun invoke(...)` per class
- All business logic lives here: budget view computation, FX conversion, virtual entry generation, conflict detection

### `ui/`
Jetpack Compose screens and `@HiltViewModel`s.
- Each feature folder contains exactly: `<Feature>Screen.kt`, `<Feature>ViewModel.kt`, `<Feature>UiState.kt`
- `components/` subfolder holds composables used only within that feature
- Composables shared across two or more features go in `ui/components/` (create this folder when the second use case appears)
- ViewModels expose a single `StateFlow<UiState>`; no repository or Firestore references inside any composable

### `workers/`
`WorkManager` workers for background tasks: health sync (every 4 hours, primary device only), nudge notifications, action reminders, subscription renewals.

### `di/`
Single `AppModule.kt` wires the full Hilt graph: `ViewModel ← UseCase ← Repository ← FirebaseManager`.

## Key Domain Concepts

**Budget Views** — the app presents two distinct views of spending, toggled in the UI (default: Budget View):
- **Cash View**: all expenses as-is, subscription lump payments at face value
- **Budget View**: subscription lump payments are hidden; virtual monthly amortised entries are injected at render time from subscription documents — these virtual entries are **never stored** in Firestore

**BYOF (Bring Your Own Firebase)**: users supply their own Firebase project credentials during onboarding. Credentials are stored in `EncryptedSharedPreferences`. A second device onboards by scanning a QR code generated by the primary device.

**Primary Device**: stored as `primaryDeviceId` in the `profile` document. Only the primary device writes `HealthSnapshot` documents and fires scheduled notifications (WorkManager jobs check this before firing).

**Conflict Resolution**:
- `Expense` and `Subscription`: manual resolution via version-checked Firestore transactions; conflicts stored in `/conflicts/{id}` and surfaced in the Conflicts screen
- `ActionLog` and `DiaryEntry`: last-write-wins (plain `.set()`, no version field)

**FX Conversion**: `amountINR = Math.round(amount × rate)` — always a whole integer. Original `amount`, `currency`, `fxRateUsed`, and `fxDate` are always preserved on the document. One Frankfurter API call per currency pair per calendar day; subsequent expenses reuse the in-memory cached rate.

**ActionLog IDs** are deterministic: `{actionId}_{date}` — enforces one log per action per day and makes upserts idempotent.

**Receipt scanning**: image is base64-encoded → sent to `gemini-1.5-flash` → structured JSON parsed → pre-fills expense form → image is **immediately discarded** and never stored.

## Invariants (Never Violate)

- Budget View virtual entries are computed at render time, never persisted to Firestore
- Receipt images are never stored anywhere
- Health data is written to Firestore only from the primary device
- Expense and Subscription conflicts must never be auto-resolved — require explicit user choice
- `domain/` has zero Android or Firebase imports
- FX rates are never written to disk
